# 安装

## 安装 Alauda 应用服务基础组件

::: tip 可选部署
该插件是 Alauda 应用服务图形界面的基本组成部分；如果不需要 Web 控制台，可以省略它。
:::

### 先决条件

1. 下载与您的平台匹配的 Alauda 应用服务基础组件。
2. 使用平台的应用列表功能，在 `global` 集群中列出 Alauda 应用服务基础组件。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，转到 **市场** > **集群插件**。
    2. 选择 `global` 集群。
    3. 点击 **Alauda 应用服务基础组件** 插件旁边的操作按钮 > **安装**。
  </Tab>
</Tabs>

## 安装 Alauda 应用服务 RDS 框架

::: tip 可选部署
该插件是 Alauda 应用服务图形界面的基本组成部分；如果不需要 Web 控制台，可以省略它。
:::

### 先决条件

1. 下载与您的平台匹配的 Alauda 应用服务 RDS 框架。
2. 使用平台的应用列表功能，在任何希望使用 Alauda 应用服务图形功能的集群中列出 Alauda 应用服务 RDS 框架。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 登录平台并转到 **平台管理** 页面。
    2. 在左侧导航栏中，选择 **市场** -> **操作中心** 进入 **OperatorHub** 页面。
    3. 查找 **Alauda 应用服务 RDS 框架**，点击 **安装**，进入部署页面。

    配置参数：

    | **参数**               | **推荐配置**                                                                                                                                 |
    | :--------------------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
    | **升级策略**           | **手动**：当 Operator Hub 中有新版本时，需要手动确认以将 Operator 升级到最新版本。                                                               |
    | **部署模式**           | **集群**：集群下的所有命名空间共享一个 Operator 来创建和管理实例，从而减少资源消耗。                                                          |
    | **频道**               | 默认频道为 **stable**。                                                                                                                      |
    | **命名空间**           | 选择 **推荐命名空间**：如果没有现有命名空间，则会自动创建。                                                                                |

    4. 在 **安装 Operator** 页面，选择 **默认配置**，然后点击 **安装** 完成 **Alauda 应用服务 RDS 框架** 的部署。
  </Tab>
</Tabs>

## 安装 Alauda Kafka 流服务插件

::: tip 可选部署
该插件管理版本为 `3.x` 的 Kafka 实例；如果需要管理 `2.x` 的实例，则必须额外安装 Alauda Kafka 2 插件。
:::

### 先决条件

1. 下载与您的平台匹配的插件安装包。
2. 使用平台的应用列表功能，在任何希望使用 Alauda Kafka 流服务功能的集群中列出该插件。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 登录平台并转到 **平台管理** 页面。
    2. 在左侧导航栏中，选择 **应用商店管理** -> **操作员** 进入 **操作中心** 页面。
    3. 查找 **Alauda Kafka 流服务**，点击 **安装**，进入部署页面。

    配置参数：

    | **参数**               | **推荐配置**                                                                                                                                 |
    | :--------------------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
    | **升级策略**           | **手动**：当 Operator Hub 中有新版本时，需要手动确认以将 Operator 升级到最新版本。                                                               |
    | **安装模式**           | **集群**：集群下的所有命名空间共享一个 Operator 来创建和管理实例，从而减少资源消耗。                                                          |
    | **频道**               | 默认频道为 **stable**。                                                                                                                      |
    | **安装位置**           | 选择 **推荐命名空间**：如果没有现有命名空间，则会自动创建。                                                                                |

    4. 在 **安装 Operator** 页面，选择 **默认配置**，然后点击 **安装** 完成部署。
  </Tab>
</Tabs>
