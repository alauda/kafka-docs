# 监控与告警

内嵌 Grafana 面板中的监控数据可用于从资源、性能等方面进行 Kafka 监控与告警，且支持配置通知策略。

## 监控

平台默认收集 Kafka 的资源、性能及容量相关常用监控指标。实例的 **监控** 页签中提供了各指标的实时监控数据。

<table border="">
    <tr>
        <th style={{width: '100px'}}>类别</th>
        <th>指标</th>
    </tr>
    <tr>
        <td><b>资源监控</b></td>
        <td>内存使用量、CPU 使用量、JVM 内存使用量等</td>
    </tr>
    <tr>
        <td><b>性能监控</b></td>
        <td>生产者请求速率、传入信息速率等</td>
    </tr>
    <tr>
        <td><b>其他</b></td>
        <td>日志大小等</td>
    </tr>
</table>

## 告警

支持使用平台通用的告警功能。进入 Container Platform 的 **告警 > 告警策略** 页面，可为 Kafka 创建以下告警对应的告警策略。

<table border="">
    <tr>
        <th style={{width: '120px'}}>类型</th>
        <th>监控对象</th>
    </tr>
    <tr>
        <td><b>指标告警</b></td>
        <td>预置的资源指标，且仅针对容器组层面，例如 容器组的内存使用率 。</td>
    </tr>
    <tr>
        <td><b>日志告警</b></td>
        <td>Kafka 日志。</td>
    </tr>
    <tr>
        <td><b>事件告警</b></td>
        <td>事件原因，即 Kafka 有状态副本集转换到当前状态的原因，例如：BackOff、Pulling、Failed。</td>
    </tr>
    <tr>
        <td><b>自定义告警</b></td>
        <td>Kafka 性能指标。</td>
    </tr>
</table>

**配置告警策略**：为启用告警，请先在 Container Platform 中创建告警策略。告警策略描述了您希望监控的对象、在什么情况下希望收到告警，以及如何被通知相关告警。

**处理告警**：当界面提示有处于告警状态的策略时，请及时处理。

**绑定通知策略**：除了查看实时提醒外，平台还支持将告警信息以邮件、短信等方式发送给相关人员，通知其采取必要的措施解决问题或避免故障发生。通知策略需在平台管理中设置。
