# 发布说明

## v4.0.0

### 新增和优化的功能

#### 组件与平台解耦

Alauda Streaming Services for Kafka 的生命周期已与平台解耦，这意味着当客户需要某个功能或修复 Bug 时，Kafka 团队可以立即发布，而无需等待平台一起发布。

#### 组件详情页面的实时日志

Kafka 详情页面的日志已被替换为实时日志，能够即时更新，方便故障排除和问题分析。

#### 通过 LoadBalancer 实现 Kafka 的外部访问

现在支持通过 LoadBalancer 进行 Kafka 访问。在创建实例时，用户可以选择通过 LoadBalancer 类型的服务将服务暴露到外部。

与 NodePort 相比，LoadBalancer 提供更好的高可用性、性能，并支持标准端口曝光。建议在生产环境中使用 LoadBalancer 方式。

`限制`

使用 LoadBalancer 时，请确保集群有可用的外部地址池。我们建议使用 MetalLB 为集群创建外部地址池。
