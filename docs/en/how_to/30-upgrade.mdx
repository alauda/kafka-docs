# Patch Version Upgrade

With the necessity of vulnerability fixes or new feature releases, patch version upgrades may occur for the middleware. During the patch version upgrade process, Pods will be restarted one by one, which may cause a temporary interruption in services. Therefore, it is recommended to select an appropriate time window for the upgrade. When a patch version is available for upgrade, it is advisable to upgrade as soon as possible to apply the latest fixes for the components.

## Set Patch Version Upgrade Strategy

### Operation Steps

<Tabs>
  <Tab label="Web Console">
    1. Go to the instance details page.

    2. Click **Patch Version Upgrade Strategy**.

    3. Choose **Manual** or **Automatic**.

    4. Click **Update** to complete the settings.
  </Tab>
</Tabs>

## Patch Version Upgrade

### Prerequisites

When a patch version is available, the instance list page will display an upgrade icon, and an upgrade prompt will appear on the instance details page.

### Operation Steps

<Tabs>
  <Tab label="Web Console">
    1. Go to the instance details page.

    2. Click **Upgrade Now** next to the upgrade prompt.

    3. Read the upgrade prompt to understand the version changes for the components and plugins.

    4. Click **Upgrade Now**.

    5. Wait for the instance status to change to Running, indicating a successful upgrade.
  </Tab>
</Tabs>

## Batch Patch Version Upgrade

### Prerequisites

User must have platform administrator permissions.

### Operation Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation bar, click **Operators**.

    2. Select the required **operator**, and go to the operator details page.

    3. Select the instances that need to be upgraded, and click **Batch Upgrade**.

    4. Read the upgrade prompt to understand the version changes for the components and plugins.

    5. Click **Upgrade Now**.

    6. Wait for the instance status to change to Running, indicating a successful upgrade.
  </Tab>
</Tabs>
