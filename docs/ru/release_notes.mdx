# Примечания к релизу

## v4.0.0

### Новые и оптимизированные функции

#### Разделение компонентной платформы

Жизненный цикл Alauda Streaming Services для Kafka был отделен от платформы, что означает, что когда клиенту нужна определенная функция или исправление ошибки, команда Kafka может выпустить обновление немедленно, не дожидаясь совместного релиза платформы.

#### Логи в реальном времени на странице деталей компонента

Логи на странице деталей Kafka были заменены логами в реальном времени, которые обновляются мгновенно, что упрощает устранение неполадок и анализ проблем.

#### Внешний доступ через LoadBalancer для Kafka

Теперь поддерживается доступ к Kafka через LoadBalancer. При создании экземпляра пользователи могут выбрать возможность внешнего выставления служб через сервис типа LoadBalancer.

По сравнению с NodePort, LoadBalancer обеспечивает лучшую высокую доступность, производительность и поддержку стандартного выставления портов. Рекомендуется использовать метод LoadBalancer в производственных средах.

`Ограничения`

При использовании LoadBalancer убедитесь, что в кластере есть доступные пул адресов для внешнего доступа. Мы рекомендуем использовать MetalLB для создания пулов внешних адресов для кластера.
