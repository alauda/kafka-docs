# Установка

## Установка Alauda Container Platform Data Services Essentials

::: tip Дополнительное развертывание
Плагин является фундаментальным компонентом представления Data Services; его можно пропустить, если веб-консоль не требуется.
:::

### Предварительные требования

1. Загрузите Alauda Container Platform Data Services Essentials, соответствующий вашей платформе.
2. Используйте функцию отображения приложений платформы, чтобы отобразить Alauda Container Platform Data Services Essentials в кластере `global`.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. В левом навигационном меню перейдите в **Marketplace** > **Cluster Plugins**.
    2. Выберите кластер `global`.
    3. Нажмите кнопку операции рядом с плагином **Alauda Container Platform Data Services Essentials** > **Установить**.
  </Tab>
</Tabs>

## Установка Alauda Container Platform Data Services RDS Framework

::: tip Дополнительное развертывание
Плагин является фундаментальным компонентом представления Data Services; его можно пропустить, если веб-консоль не требуется.
:::

### Предварительные требования

1. Загрузите Alauda Container Platform Data Services RDS Framework, соответствующий вашей платформе.
2. Используйте функцию отображения приложений платформы, чтобы отобразить Alauda Container Platform Data Services RDS Framework в любом кластере, где вы хотите использовать представление Data Services.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Войдите в платформу и перейдите на страницу **Администратор**.
    2. В левом навигационном меню выберите **Marketplace** -> **OperatorHub**, чтобы войти на страницу **OperatorHub**.
    3. Найдите **Alauda Container Platform Data Services RDS Framework**, нажмите **Установить**, и перейдите на страницу развертывания.

    Параметры конфигурации:

    | **Параметр**           | **Рекомендуемая конфигурация**                                                                                                                       |
    | :--------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Стратегия обновления** | **Ручное**: Когда в OperatorHub появляется новая версия, требуется ручное подтверждение для обновления оператора до последней версии.      |
    | **Режим развертывания** | **Кластер**: Все пространства имен в кластере используют одного оператора для создания и управления экземплярами, что приводит к снижению потребления ресурсов. |
    | **Канал**             | Канал по умолчанию - **stable**.                                                                                                                     |
    | **Пространство имен** | Выберите **Рекомендуемое пространство имен**: автоматически создается, если оно отсутствует.                                                       |

    4. На странице **Установить оператор** выберите **Конфигурация по умолчанию**, затем нажмите **Установить**, чтобы завершить развертывание **Alauda Container Platform Data Services RDS Framework**.
  </Tab>
</Tabs>

## Установка плагина Alauda Streaming Service for Kafka

:::tip Дополнительное развертывание
Этот плагин управляет экземплярами Kafka для версий `3.x`; если вам нужно управлять экземплярами `2.x`, дополнительно необходимо установить плагин Alauda Build of Kafka 2.
:::

### Предварительные требования

1. Загрузите установочный пакет плагина, соответствующий вашей платформе.
2. Используйте функцию отображения приложений платформы, чтобы отобразить плагин в любом кластере, где вы хотите использовать возможности Alauda Streaming Service for Kafka.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Войдите в платформу и перейдите на страницу **Администратор**.
    2. В левом навигационном меню выберите **Marketplace** -> **Operator**, чтобы войти на страницу **OperatorHub**.
    3. Найдите **Alauda Streaming Service for Kafka**, нажмите **Установить**, и перейдите на страницу развертывания.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                       |
    | :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Стратегия обновления**  | **Ручное**: Когда в OperatorHub появляется новая версия, требуется ручное подтверждение для обновления оператора до последней версии.      |
    | **Режим установки**      | **Кластер**: Все пространства имен в кластере используют одного оператора для создания и управления экземплярами, что приводит к снижению потребления ресурсов. |
    | **Канал**                | Канал по умолчанию - **stable**.                                                                                                                     |
    | **Место установки**      | Выберите **Рекомендуемое пространство имен**: автоматически создается, если оно отсутствует.                                                       |

    4. На странице **Установить оператор** выберите **Конфигурация по умолчанию**, затем нажмите **Установить**, чтобы завершить развертывание.
  </Tab>
</Tabs>
